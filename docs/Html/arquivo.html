Satisfação do Cliente
Análise Preditiva de NPS

Apresentado por: Tuany Fortunato do Carmo

O Desafio: Escala vs. Satisfação

Nossa operação cresceu rápido, mas a satisfação ficou pelo caminho. O grande problema é que somos reativos.


  • Descobrimos o problema só após o encerramento da jornada.
  • Quando o NPS chega, o cliente já teve a experiência ruim.
  • Não estamos identificando os sinais de alerta cedo o bastante.
Pergunta Central: Como antecipar a insatisfação antes que ela se torne uma nota baixa?

E quais fatores operacionais influenciam diretamente para que os cliente sejam detratores?

Termômetro Atual da Satisfação

O cenário é crítico:

  • 74% da nossa base são Detratores.
  • Apenas 8% são Promotores.
  • Nota média de 4,3: estamos abaixo do padrão de mercado para e-commerce.
Mensagem: A experiência atual do cliente não está adequada, e precisamos entender os motivos para agir pró-ativamente.

Logística: O Maior Destruidor de Valor

O dado é implacável: o atraso na entrega é o fator que mais empurra o cliente para a detração.


  • 1 dia de atraso: queda drástica na nota.
  • 3+ dias de atraso: o cliente vira detrator com quase 100% de certeza.
O atraso logístico é o "ponto de ruptura" da nossa experiência.

SAC: Onde o Problema se agrava

O cliente não quer ser atendido; ele quer que o processo funcione.


  • Quanto mais o cliente aciona o SAC, menor é a nota.
  • 3+ contatos indicam que o problema não foi resolvido na primeira vez.
Resolução no 1º contato com o cliente é vital para salvar o NPS.

CSAT Interno vs. Realidade (NPS)

A percepção interna do atendimento (CSAT) pode ser enganosa.


  • Muitas vezes o atendente é educado (CSAT alto), mas o processo falhou (NPS baixo).
  • Um bom atendimento não apaga uma entrega ruim.
Não podemos nos confortar com métricas de "simpatia" se a operação falha.

A Solução: Inteligência Artificial

Proposta: Modelo de Classificação Binária.


Em vez de esperar o cliente responder, vamos prever quem está em risco com base em:

  • Dias de atraso atualizados em tempo real.
  • Número de tentativas de entrega.
  • Reincidência de contatos no SAC.

Saída do Modelo:

"Cliente em Risco: SIM/NÃO"

O objetivo é munir o time de CS com uma lista de ação ANTES da pesquisa.

Plano de Ação: Próximos 30 Dias

1. Alerta de Atraso Crítico

Priorizar pedidos com 2+ dias de atraso para entrega "super expressa".

2. Blindagem de Reputação

Ação proativa (cupom/estorno) para clientes mapeados como "Em Risco".

3. Eficiência no SAC

Escalar automaticamente chamados recorrentes de um mesmo cliente.

4. Integração de Dados

Unificar visão de Logística e Atendimento em um único dashboard. Para acompanharmos o desempenho após as ações implementadas.

Transformar dados em ação é o caminho para recuperar a confiança.
